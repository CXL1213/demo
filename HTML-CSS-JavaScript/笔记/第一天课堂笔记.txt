1.当前的系统结构有哪些？
	C/S结构
		Client / Server (客户端/服务器端)
		C/S结构的系统，我们在使用的时候是需要安装特定的客户端软件的，例如：
			QQ、微信...
		C/S结构的系统大部分都集成在客户端软件上，服务器需要发送的数据量
		很小，所以C/S结构的系统速度快。

		C/S结构的系统有什么缺点？
			第一：需要安装特定的客户端软件。
			
			第二：不好升级，不好维护，后期升级比较麻烦。
			因为只要系统升级，所有的客户端软件都需要重新安装。
			并且对于一些特殊的软件俩说，一般人还安装不了，需要专业人员进行环境安装配置。
		
		注重用户体验的系统一般都是C/S结构。
		要求画质好看，要求速度高。
		所以，一般都是娱乐性的系统都是基于C/S结构。
		例如：3D游戏。

	B/S结构
		Browser / Server （浏览器 / 服务器端）
		浏览器软件作为客户端，不需要再安装其他特定的软件，打开浏览器，直接在浏览器的地址栏上输入地址即可访问服务器。

		B/S 结构的系统，所有的数据都是在服务器端，在客户端上只有一个普通的浏览器软件，
		所有的数据都是从服务器上过来的，所以B/S结构的系统速度慢。

		B/S 结构的优点？
			所有的数据都是集成在服务器上，所以升级的时候只需要升级一个服务器，客户端的浏览器还是那个浏览器。B/S结构升级维护成本低。

		公司/企业内部使用的一些系统需要很好看吗？需要体验特别好吗？
		不需要，企业内部使用的系统其实就是完成数据的维护即可。

		一般企业级的项目都是基于B/S结构的。
		例如：OA办公系统，CRM客户管理系统，DRP进销存系统....
		管理系统都是基于B/S结构。

3.什么是HTML？
	Hyper text Markup Language:超文本标记语言
	超文本：不是普通文本，例如图片、声音、视频等流媒体数据。（HTML支持流媒体）
	标记语言：标记语言一般都是由一对一对的标签组成。
		成对儿出现：
			<html></html>
			以上就是HTML的根标签。
			HTML 中大部分的标签都是由开始标签和结束标签组成。都是成对出现。
			你编写代码的时候也要成对编写。养成好习惯。
			所有的结束标签开始的第一个字符都是正斜杠：/
